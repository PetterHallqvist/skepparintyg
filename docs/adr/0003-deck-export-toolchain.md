# ADR-0003: Deck export toolchain (Anki .apkg + Quizlet TSV)

_2026-07-16 · accepted · continuous record · SPEC §37 (M4)_

## Decision

Deck exports are built in an offline pipeline (`/pipeline/decks`), not at
request time:

- **Quizlet:** UTF-8 **TSV** (tab between front/back, newline between cards) —
  the layout Quizlet's importer accepts (SPEC §37.2). Generated by a pure
  function; also served in-memory by the download route.
- **Anki:** **`.apkg`** built entirely in **TypeScript/JS** — `sql.js`
  (pure-wasm SQLite, no native dependency) writes `collection.anki2`, and
  `fflate` zips it. No Python `export_anki.py`; the toolchain stays uniform with
  the rest of the repo. Both are **devDependencies** — the builder never ships in
  the app bundle.

Determinism is a hard requirement (SPEC §37.6): every id and timestamp is
derived from deck content (never `Date.now`), and zip entry mtimes are fixed, so
a deck always produces byte-identical output. `build.mjs` builds each `.apkg`
twice and fails if the bytes differ.

## Rationale

`sql.js` produces a genuinely importable `.apkg` without a native SQLite build
or a Python step, so the pipeline runs anywhere Node runs (local, CI). A pure
TSV path covers Quizlet with zero dependencies and is the always-working
download in demo mode.

## Consequences

- The binary `.apkg` is a build artifact (`pipeline/decks/dist/`, gitignored),
  regenerated by `pnpm build:decks`. The download route serves it from disk and
  returns 501 if it hasn't been built; TSV is always available in-memory.
- Text snapshots (`.tsv` + import guide) and per-deck manifests (with SHA-256
  checksums) are committed under `pipeline/decks/{snapshots,manifests}` as the
  reviewable record.
- Entitlement is enforced fail-closed in the route: the free lead-magnet deck
  downloads now; paid decks return 403 until real entitlements land (M6, §37.7).
- No invasive DRM; no per-card purchaser watermark (§37.7).
